# RestoPanel · Estrategia de marca, copy y SEO

## 1. Nombres de módulos (branding de features)

Cada módulo tiene un nombre comercial propio que se usa en la landing, en el panel y en materiales de venta. Los nombres siguen el patrón `Resto + [concepto]` para mantener coherencia de marca.

| Módulo | Nombre comercial | Función |
|---|---|---|
| RestoBookings | Centralización de reservas | Unifica reservas de web, Google, Instagram, Facebook, WhatsApp, teléfono y walk-ins en un solo libro digital |
| RestoFloor | Asignación inteligente de mesas | Motor de sala que combina plano, zonas, turnos y duración media para maximizar aforo |
| RestoGuard | Control de no-shows | Reconfirmación automática, tarjeta como garantía opcional, prepago de experiencias |
| RestoCRM | CRM y base de datos propia | Historial de visitas, ticket medio, preferencias, campañas segmentadas por SMS/email/WhatsApp |
| RestoNight | Ocio nocturno y eventos | Entradas online, control de accesos, listas de invitados, zonas VIP, botellas, packs |
| RestoQueue | Cola virtual y listas de espera | Cola con notificación por SMS/WhatsApp cuando la sala está llena |
| RestoAnalytics | Analítica y reporting | KPIs en tiempo real: ocupación, ticket medio, recurrencia, no-shows, rendimiento por turno y canal |

### Sección 7 — FAQs

**¿Qué es RestoPanel?**
RestoPanel es un software en la nube que centraliza la gestión de reservas, mesas, clientes, eventos y analítica para restaurantes, discotecas, clubs, beach clubs y hoteles. Todo funciona desde un único panel accesible desde cualquier dispositivo con navegador. No necesitas instalar nada: creas tu cuenta, configuras tu sala y empiezas a trabajar.

**¿Qué tipo de negocios pueden usar RestoPanel?**
RestoPanel está diseñado para tres grandes perfiles: restaurantes independientes y grupos hosteleros; negocios de ocio nocturno (discotecas, clubs, beach clubs, festivales); y hoteles con F&B, rooftops, bares y restaurantes internos. Cada perfil puede activar los módulos que necesita y dejar desactivados los que no use.

**¿Necesito formación para usarlo?**
No. La interfaz está pensada para personal de sala y de puerta que atiende clientes en horas de máxima demanda. El onboarding es guiado: configuramos contigo tu plano de sala, tus turnos y tus canales en la primera sesión. Si tu equipo sabe usar un WhatsApp, sabe usar RestoPanel.

**¿Cómo se integra con mis canales actuales?**
RestoPanel centraliza reservas que llegan de tu web propia, Google Maps, Instagram, Facebook, WhatsApp y teléfono. La disponibilidad se sincroniza en tiempo real entre todos los canales, así que nunca se acepta una reserva para una mesa que ya no está libre. Los walk-ins también se registran en el mismo panel.

**¿En qué idiomas está disponible la interfaz?**
La interfaz principal está en español (tono neutro profesional, apto para España y Latinoamérica). El equipo de soporte atiende en español. Próximamente: inglés y portugués de Brasil para locales con personal internacional o turista extranjero.

**¿Cómo se gestionan los datos y la privacidad de mis clientes?**
Los datos de tus clientes viven en tu propia base de datos dentro de RestoPanel. Nunca se comparten con terceros, nunca se venden a OTAs ni a plataformas de reservas externas. Tú decides qué datos guardas, qué campañas lanzas y cuándo las lanzas. Cumplimos con la normativa europea de protección de datos (RGPD).

**¿Puedo usarlo sólo para eventos o sólo para reservas de restaurante?**
Sí. RestoPanel es modular. Si solo necesitas gestionar reservas de restaurante, activas RestoBookings, RestoFloor y RestoCRM. Si además haces eventos o tienes ocio nocturno, activas RestoNight para entradas, listas y zonas VIP. Si solo quieres el CRM para fidelizar clientes existentes, también puedes usarlo independientemente.

**Sinónimos integrados**:
- reservas digitales, motor de reservas, libro de reservas online, panel de control de reservas, software de hostelería

### Estructura SEO on-page

- **H1**: "Llena más mesas. Pierde menos reservas. Conoce a cada cliente." (1 solo, con keywords implícitas)
- **H2**: 7 secciones con H2 únicos y descriptivos
- **H3**: Cada módulo y cada caso de uso tiene su H3
- **FAQ**: 7 preguntas con schema JSON-LD `FAQPage` (rich snippets en Google)
- **Schema**: `SoftwareApplication` + `FAQPage` (datos estructurados para Google)
- **Meta title**: 60 chars, con keyword principal
- **Meta description**: 155 chars, con keywords y CTA implícito
- **Canonical**: `/landing`
- **OpenGraph**: title, description, locale es_ES

## 5. Diferenciación frente a competidores

| Mensaje genérico de competidores | Mensaje de RestoPanel |
|---|---|
| "Plataforma de reservas avanzada" | "Llena más mesas. Pierde menos reservas." |
| "Solicita una demo" | "Crear cuenta gratis" (prueba directa) |
| "Conecta con miles de comensales" | "Datos 100% tuyos. Sin intermediarios." |
| "Gestión de reservas" | "Todo lo que pasa en tu sala, en un solo panel" |
| "Reduce no-shows" | "Hasta -35% de ausencias con reconfirmación automática" |
| "CRM integrado" | "CRM propio: historial, ticket medio y campañas a tus clientes" |
| "Para restaurantes" | "Para restaurantes, discotecas, clubs, beach clubs y hoteles" |

**Principio diferenciador**: cada frase apunta a un beneficio medible (ocupación, ingresos, tiempo ahorrado), no a una feature técnica. El dueño del negocio entiende "más mesas llenas" antes que "plataforma cloud escalable".
